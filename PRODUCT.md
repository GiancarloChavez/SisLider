# SisLider — Product Context

## Product Purpose
Internal academic management system for a small educational institute. Used exclusively by one secretary to manage student enrollment, attendance, class schedules, and monthly payments.

## Register
product

## Users
Single internal user: a school secretary who operates the system daily on a desktop computer in an office environment. No technical background required. The interface must be efficient for repetitive daily tasks (registering payments, checking attendance, enrolling students).

## Brand Tone
Professional, calm, trustworthy. No playfulness. The system should feel like a reliable tool — not a consumer app. Think of tools like Linear or Notion: purposeful, quiet, fast.

## Anti-References
- Colorful SaaS dashboards with excessive gradients and celebrations
- Consumer app aesthetics (rounded corners everywhere, bubbly UI)
- Outdated admin panels (Bootstrap gray tables, heavy borders)
- Anything that feels like a template

## Strategic Principles
- Efficiency over decoration: every element serves a task
- Calm hierarchy: the most urgent information surfaces naturally
- Minimal chrome: the interface should recede, the data should lead
- One action per moment: no decision paralysis
